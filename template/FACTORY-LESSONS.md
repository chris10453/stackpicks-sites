# Factory lessons

Append one entry after every staging draft cycle. Each entry records what broke, what improved, and the template delta applied before the next cycle.

## Cycle 1
- Niche/product class: pending selection gate
- What broke: pending first factory build
- What improved: factory loop requires one niche at a time, competitor teardown before build, and staging verifier exit 0 before handoff
- Template delta: added this lessons ledger; next cycle must update the canonical template before another site

## Cycle 2 — Amazon-class / content-cluster pivot
- Niche/product class: home-office gear content cluster
- What broke: the shared public verifier is intentionally public-domain/ClickBank-shaped; it rejects an internal staging sitemap and has no draft-aware program-placeholder state.
- What improved: the draft keeps every commercial link suppressed, uses a visible no-qualifying-offer state, and records `AMZN-TAG-PENDING` only in draft notes/registry rather than presenting a usable link.
- Template delta: draft pages now require a staging-only banner, explicit program gate, no-form/no-stars/no-claims rules, and a no-qualifier state. Claude must add a draft-aware verifier rule; Hermes does not edit the shared verifier.

## Cycle 3 — Etsy printable pivot
- Niche/product class: printable planners and digital templates
- What broke: marketplace roundups make it easy to imply file quality or buyer proof without inspecting the actual listing.
- What improved: each card names the decision criteria (format, print specs, licensing) and suppresses offers until Awin/Etsy approval and listing-fact verification.
- Template delta: digital products need format/license/preview checks before monetization, not just price and merchant checks.

## Cycle 4 — print-on-demand pivot
- Niche/product class: maker gifts and practical hobby apparel
- What broke: POD pages can hide fulfillment, personalization, shipping, and returns behind lifestyle copy.
- What improved: every draft card makes those policy facts explicit and keeps `POD-TAG-PENDING` in the registry/draft notes only.
- Template delta: POD selection requires fulfillment, personalization, delivery, and returns fields before any live CTA.

## Cycle 5 — stranger batch 3 addendum
- Niche/product class: all content-filled drafts, including photo-driven categories
- What broke: two template families made some drafts look broken; the styled family made its unfinished state too prominent; product-photo categories had no safe pre-approval visual path.
- What improved: all factory drafts now use the styled family with per-site palette variation; the draft banner is a single dismissible top bar and is removed by the publish flag; photo-driven drafts remain text-first until approved program tooling exists.
- Template delta: never scrape Amazon/Etsy/merchant photos before approval. Use CC0/Openverse/Wikimedia/own generated illustrative assets later, or ship text-first. Keep all commercial links suppressed until the gate receipt and listing facts pass.
